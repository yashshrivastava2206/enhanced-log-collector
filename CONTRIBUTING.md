# Contributing to Enhanced Log Collector

Thank you for considering contributing to the Enhanced Log Collector!

## Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/your-username/enhanced-log-collector.git
   cd enhanced-log-collector
   ```
3. Create a development environment:
   ```bash
   make dev-setup
   ```

## Making Changes

1. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Make your changes following our coding standards:
   - Use 4 spaces for indentation
   - Comment complex logic
   - Follow existing naming conventions
   - Add tests for new features

3. Test your changes:
   ```bash
   make test
   ```

4. Update documentation:
   - Update README.md if needed
   - Add examples to `examples/`
   - Update CHANGELOG.md

5. Commit your changes:
   ```bash
   git add .
   git commit -m "feat: add your feature description"
   ```

## Commit Message Format

Use conventional commits:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `test:` Test additions or changes
- `refactor:` Code refactoring
- `perf:` Performance improvements
- `chore:` Build/tooling changes

## Pull Request Process

1. Update the VERSION file if needed
2. Ensure all tests pass
3. Update documentation
4. Submit PR with clear description
5. Wait for review from maintainers

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what is best for the project
- Show empathy towards other contributors
